# Security Policy

Last updated: February 15, 2026

This document describes how to report security issues for OGS Launcher.

## Reporting a Vulnerability

Please report security issues privately. Do not open a public issue for security vulnerabilities.

When reporting, include:
- A clear description of the issue and impact.
- Steps to reproduce, including any sample files or project structure.
- Affected versions or commit hashes if known.
- Any suggested remediation ideas.

## Response Expectations

We will acknowledge receipt as soon as practical and coordinate a fix. Timelines depend on severity and scope.

## Supported Versions

Security fixes are applied to the current main branch. Backports are evaluated case-by-case.
